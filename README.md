# Setup Portfolio — Junior Growth Marketing Specialist at 100Hires

This repository documents the tools and setup process completed as part of my application for the Junior Growth Marketing Specialist role at 100Hires.

---

## Tools Installed

- **Cursor IDE** — An AI-powered code editor built on top of VS Code, designed to work with AI assistants for faster development and content creation.
- **Claude Code Extension** — An extension inside Cursor that provides access to Anthropic's Claude AI directly within the editor for writing, debugging, and brainstorming.
- **Codex Extension** — An extension inside Cursor that connects to OpenAI's Codex, enabling AI-assisted coding and content generation.
- **Git & GitHub** — Version control system and cloud-based platform for managing and sharing code repositories.

---

## Steps Completed

1. **Downloaded and installed Cursor IDE** — Visited cursor.com, downloaded the Windows installer, granted the necessary permissions, and completed the installation.

2. **Installed Claude Code extension** — On the initial Cursor screen, there was no visible option to add extensions. Navigated to the full editor window, opened the Extensions panel from the View tab, searched for "Claude Code," installed it, and completed the authentication process.

3. **Installed Codex extension** — Opened the Extensions panel again, searched for "Codex," installed it, and completed the login and authentication process separately.

4. **Logged into GitHub** — Went to github.com and signed in using Google authentication with my existing account.

5. **Created a public repository** — On the GitHub homepage, clicked "New" next to the Top Repositories section. Named the repository `KartikProject`, added the description "Sample Project for 100Hires," and set it to public.

6. **Cloned the repository in Cursor** — Copied the repository URL from GitHub. In Cursor, created a local project folder named "100Hires Sample Project." Opened the terminal via the View tab and ran `git clone https://github.com/koga1212/KartikProject.git` to clone the repo locally.

7. **Created this README.md** — Opened the cloned repository folder in Cursor and created this file to document the full setup process.

## Issues Encountered and How I Solved Them

### 1. Extensions panel not visible on initial screen
On first opening Cursor, I could not find the option to install extensions. The initial welcome screen did not display it. I explored the editor interface and found the Extensions panel under the View tab. This resolved the issue.

### 2. Opening a GitHub repository in Cursor
After creating the repository on GitHub, I was unsure how to open it locally in Cursor. I asked Claude for guidance, followed the steps to copy the repo URL, clone it via the terminal, and open the folder in Cursor. The process worked without further issues.
